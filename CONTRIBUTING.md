# Вклад в проект (Contributing)

Спасибо за интерес к проекту! Ниже — краткое руководство по участию в разработке.

## Как сообщить об ошибке

1. Убедитесь, что похожий issue ещё не создан — поищите в [Issues](../../issues).
2. Создайте новый issue с подробным описанием:
   - Что вы делали?
   - Что ожидали увидеть?
   - Что произошло на самом деле?
   - Версии Python, Django, ОС.

## Как предложить улучшение

1. Откройте issue с тегом `enhancement` и опишите идею.
2. Дождитесь обратной связи, прежде чем начинать реализацию.

## Процесс Pull Request

1. Форкните репозиторий.
2. Создайте ветку от `main`:
   ```bash
   git checkout -b feature/my-feature
   ```
3. Внесите изменения и зафиксируйте их с понятным сообщением:
   ```bash
   git commit -m "feat: add my feature"
   ```
4. Убедитесь, что проект запускается без ошибок:
   ```bash
   python3 manage.py check
   python3 manage.py test
   ```
5. Отправьте ветку в свой форк:
   ```bash
   git push origin feature/my-feature
   ```
6. Откройте Pull Request в основной репозиторий.

## Стиль кода

- Следуйте [PEP 8](https://peps.python.org/pep-0008/).
- Используйте понятные имена переменных и функций.
- Добавляйте комментарии к нетривиальному коду.

## Лицензия

Отправляя Pull Request, вы соглашаетесь с тем, что ваши изменения будут распространяться под лицензией [MIT](LICENSE).
